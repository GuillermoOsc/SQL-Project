Plan y estrategía de pruebas


Pruebas unitarias:

Verificar que todas las tablas hayan sido creadas correctamente y que todos los campos estén presentes.
Verificar que todas las claves foráneas estén funcionando correctamente y que los registros relacionados 
se estén actualizando correctamente.
Verificar que los valores de los campos se estén validando correctamente.


Pruebas de integridad de datos:

Verificar que los datos se estén actualizando correctamente en las tablas relacionadas cuando se realiza una acción,
como agregar una reserva o cancelar una reserva.
Verificar que los valores de los campos se estén validando correctamente, por ejemplo, que los precios
de las habitaciones y servicios no sean valores negativos.


Pruebas de rendimiento:
Medir el tiempo de respuesta de consultas complejas que involucren varias tablas o que devuelvan 
grandes cantidades de datos.
Verificar que la base de datos pueda manejar una gran cantidad
de datos sin degradar el rendimiento.


Pruebas de seguridad:
Verificar que la base de datos esté protegida contra ataques maliciosos, como inyecciones SQL o accesos no autorizados.
Verificar que los datos se estén almacenando de manera segura y que se estén implementando medidas de seguridad adecuadas, como el cifrado de contraseñas y la restricción de acceso a datos sensibles.
Pruebas de usuario final:
Verificar que los usuarios puedan realizar tareas comunes, como hacer reservas, cancelar reservas, agregar servicios adicionales, entre otros.
Verificar que la interfaz de usuario sea intuitiva y fácil de usar.
